# Клиентски портал — локална настройка

Бизнес изгледът (Flows, „Изпълни", прогрес, Резултат, разговорен създател) живее на
**отделен поддомейн**, а админ приложението — на основния домейн. Изолацията се прави с
два env ключа в `.env`:

```env
APP_DOMAIN=flowai.local.com            # админ приложението (routes/web.php)
CLIENT_DOMAIN=clients.flowai.local.com # клиентският портал (routes/client.php)
```

- `APP_DOMAIN` ограничава **всички** админ routes до основния домейн, така че на поддомейна
  се вижда само клиентският портал (иначе напр. `/` би отворило админ home-а).
- `CLIENT_DOMAIN` качва клиентския портал на поддомейна.

## Поддомейнът локално

Поддомейнът трябва да сочи към същия локален сървър. Добави запис в `/etc/hosts`:

```
127.0.0.1   clients.flowai.local.com
```

…и подай хоста на уеб сървъра (Valet/Nginx/Caddy/Herd), който вече обслужва
`flowai.local.com`, така че да отговаря и на `clients.flowai.local.com` (същият
document root + TLS, ако ползваш https). При Valet това обикновено е автоматично за
поддомейните на линкнат сайт.

После: `https://clients.flowai.local.com/login`.

## Fallback без поддомейн (path-префикс)

Ако не искаш да вдигаш поддомейн локално, **остави и двата ключа празни**:

```env
APP_DOMAIN=
CLIENT_DOMAIN=
```

Тогава админът остава глобален (днешното поведение), а клиентският портал минава на
path-префикс: `https://flowai.local.com/client/login`. Нищо друго не се променя.

## Вход (preview)

Входът е **временен** за preview: избор на фирма → потребител, без парола. Всяка фирма
получава служебен `owner` автоматично (при създаване в админа + миграция за вече
съществуващите). Реалната аутентикация (парола/magic-link) е следваща фаза.

## Зависимости за изпълнение и създаване

- **Изпълнение на Flow** изисква жив Horizon worker (`composer dev` или `php artisan horizon`).
  Без него картата показва приятелско съобщение.
- **Разговорният създател** изисква cloud AI с tool calling. Ползва `CLIENT_WIZARD_PROVIDER`/
  `CLIENT_WIZARD_MODEL`; ако са празни → пада на `BUILDER_ASSISTANT_*`, после на planner-а
  (ако е cloud), иначе `openai`.
- **Ниво на модела per-фирма**: админ формата за фирма има select „Ниво на модела за
  клиентски flows" (`companies.settings.model_level`, default `medium`); клиентската генерация
  чете него. Клиентът не вижда тази настройка.
