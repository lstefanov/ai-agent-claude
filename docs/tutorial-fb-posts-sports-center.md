## 2. Изтегли нужните AI модели

Отиди на [https://flowai.local/models](https://flowai.local/models)

### Задължителни модели за тестовия flow:

| Модел | Размер | Употреба |
|---|---|---|
| `todorov/bggpt` | ~2.3 GB | Текст на **български** |
| `mistral-nemo` | ~6.7 GB | JSON, анализ, image prompts |
| `phi3.5` | ~2.2 GB | QA проверка (бърз) |

### Как да изтеглиш:

1. Намери модела в списъка
2. Натисни **⬇ Изтегли**
3. Изчакай — ще видиш прогрес бар с фазата: `Изтегляне на манифест…` → `Изтегляне на слоеве…` → `100%`
4. При успех — зелена точка и бутон **▶ Тест**

### Тествай дали моделът работи:

Натисни **▶ Тест** до всеки модел. Трябва да получиш отговор за 5-15 секунди:

```
OK – BgGPT works.
```

> **Синхронизирай** — ако моделите са изтеглени но показват "недостъпен", натисни **🔄 Синхронизирай** горе вдясно.

## 4. Създай нов Flow

От страницата на фирмата натисни **＋ Нов flow**.

Попълни:

**Наименование на flow-а:**
```
Ежедневни FB постове
```

**Описание на flow-а:**
```
Генерирай ангажиращ Facebook пост на български за спортния център. 
Постът трябва да промотира дейности, мотивира хората да спортуват 
и да включва подходящи хаштагове. Постът трябва да съдържа изображение.
```

**Статус:** `Active`

**Cron разписание:**
```
0 10 * * *
```
*(изпълнява се всеки ден в 10:00 сутринта)*

## 5. Генерирай агентите с AI

Натисни **✨ Генерирай агенти с AI**

> ⏳ Изчакай **60-120 секунди** — AI анализира описанието и проектира пълния pipeline. **Не затваряй страницата.**

Трябва да се появят **6-7 агента**:

| # | Агент | Роля | Модел |
|---|---|---|---|
| 1 | `researcher_bg` | Изследва актуални теми и тенденции | BgGPT |
| 2 | `analyzer_bg` | Анализира и избира най-добрата тема | mistral-nemo |
| 3 | `content_bg` | Пише текста на поста на български | BgGPT |
| 4 | `hashtag_bg` | Генерира хаштагове (BG + EN) | mistral-nemo |
| 5 | `image_prompt_bg` | Описва изображението за ComfyUI | mistral-nemo |
| 6 | `caption_writer_bg` | Сглобява финалния пост | mistral-nemo |
| 7 | `qa_verifier_bg` | Проверява качеството (0-100) | phi3.5 |

> **Ако получиш по-малко от 5 агента** — натисни отново **✨ Генерирай агенти с AI**. AI може да върне различен брой при различни заявки.

## 6. Прегледай и запази Flow-а

### Провери моделите:

За всеки агент може да смениш модела от dropdown-а. Препоръки:
- Агенти за **български текст** → `todorov/bggpt`
- Агенти за **JSON, анализ, image prompts** → `mistral-nemo`
- **QA verifier** → `phi3.5` или `phi3:mini` (по-бързи)

> Модели с **⚠** пред името не са изтеглени. Те ще работят само ако ги изтеглиш от страницата с модели.

### Разгледай детайлите:

Натисни **▼ Покажи всичко** до всеки агент за да видиш:
- Prompt шаблона (какво точно се изпраща на модела)
- Вход / изход описание
- QA праг (само за qa_verifier)

### Запази:

Натисни **💾 Запази flow с агентите**

При успех → пренасочва към страницата на flow-а.

От страницата на flow-а натисни **▶ Стартирай сега**

> ⏳ Изпълнението отнема **2-8 минути** в зависимост от броя агенти и размера на моделите. Страницата се обновява автоматично.

**Какво се случва зад кулисите:**

```
researcher_bg   → "Фитнес тенденции, пролетни предизвикателства, груповите тренировки..."
      ↓
analyzer_bg     → "Избрана тема: Пролетно предизвикателство 30 дни"
      ↓
content_bg      → "🌟 Пролетта е тук! Готов ли си за 30-дневното предизвикателство?..."
      ↓
hashtag_bg      → "#фитнес #спорт #здравословенживот #30дни #СпортенЦентърДинамо"
      ↓
image_prompt_bg → "photorealistic modern gym, spring light, people working out, 
                   motivational atmosphere, 4K, professional photography"
                    ↓
                 ComfyUI генерира изображение (~30-60 сек на M3 Max)
                 Запазено в: storage/app/public/generated/abc123.png
      ↓

      ↓
```

## 8. Виж резултата

### В списъка с runs:

На страницата на flow-а виж секцията **Runs** — последният run трябва да е `✅ Завършен`.

Натисни **Отвори →** до run-а.

### На страницата на run-а:

Виждаш всяка стъпка поотделно:

- **Зелена точка** = успешно завършен агент
- **Червена точка** = грешка (разшири за детайли)
- **QA badge** = score на verifier-а (напр. `87%`)

Натисни върху всеки агент за да разгъне изхода му.

## 9. Настрой автоматичен scheduler

За да се изпълнява flow-ът **автоматично всеки ден в 10:00**, трябва Laravel scheduler да върви.

### Провери cron-а:

```bash
crontab -l
```

Ако не виждаш Laravel scheduler — добави го:

```bash
crontab -e
```

Добави реда:

```
* * * * * cd /Users/lub/Sites/localhost/ai-agent-claude && /opt/homebrew/bin/php artisan schedule:run >> /tmp/laravel-schedule.log 2>&1
```

> Cron-ът се изпълнява всяка минута. Laravel сам проверява кои flows трябва да стартират спрямо техния cron (`0 10 * * *`).

### Провери дали scheduler работи:

```bash
/opt/homebrew/bin/php artisan schedule:list
```

Трябва да видиш:

```
0 * * * *    SyncOllamaModelsJob
* * * * *    flows:run-scheduled
```

### Тест на scheduler ръчно:

```bash
/opt/homebrew/bin/php artisan flows:run-scheduled
```

Ако flow-ът има `status = active` и cron съвпада с текущото време — ще стартира.

## 10. Troubleshooting

### ❌ "Мрежова грешка" при генериране на агенти

```bash
# Провери дали Ollama работи
curl http://localhost:11434/api/tags

# Провери лога на генерацията
tail -f storage/logs/agent-gen.log
```

### ❌ Моделът стои на 0% при изтегляне

```bash
# Провери лога за грешки
cat storage/logs/pull-{id}.log

# Провери дали процесът работи
ps aux | grep "models:pull"
```

Ако логът показва `file does not exist` — тагът на модела е грешен. Провери на [ollama.com/library](https://ollama.com/library).

### ❌ Flow run се проваля на определен агент

1. Отвори run-а от [https://flowai.local/runs/{id}](https://flowai.local/runs)
2. Разгъни агента с червената точка
3. Виж съобщението в **⚠ Грешка** секцията
4. Провери дали моделът е наличен на страницата с модели

### ❌ Изображението не се генерира

```bash
# Провери дали ComfyUI работи
curl http://localhost:8188/system_stats

# Стартирай ComfyUI ако не работи
cd ~/ComfyUI && source venv/bin/activate
python main.py --listen 127.0.0.1 --port 8188
```

### ❌ QA verifier спира flow-а

Ако score е под прага (default 75/100), flow-ът спира. Можеш да:
1. Отвориш агента от **Flows → [flow] → агент → ✏ Редактирай**
2. Намалиш **QA праг** от 75 на 60
3. Или подобриш prompt шаблона на content агентите

### 📋 Полезни команди

```bash
# Виж всички логове в реално време
tail -f storage/logs/laravel.log

# Ръчно изпълни scheduled flows
/opt/homebrew/bin/php artisan flows:run-scheduled

# Синхронизирай Ollama модели
/opt/homebrew/bin/php artisan app:sync-ollama-models

# Провери cache (generation tokens)
/opt/homebrew/bin/php artisan cache:clear
```
